# Event Registration System - Design Guidelines

## Design Approach

**Selected Approach**: Design System-based with dual personality
- **Public Registration Form**: Inspired by Linear's clean aesthetic with Stripe's trust-building elements
- **Admin Dashboard**: Material Design principles for data-dense interfaces with clear information hierarchy

This dual approach balances user-friendliness for attendees with efficiency for administrators.

## Typography System

**Font Stack**: 
- Primary: Inter (Google Fonts) - all UI elements, forms, dashboard
- Monospace: JetBrains Mono - ticket IDs, QR data display

**Hierarchy**:
- H1: text-4xl font-bold (Page titles, "Event Registration")
- H2: text-2xl font-semibold (Section headers, "Registration Details")
- H3: text-xl font-medium (Card titles, table headers)
- Body: text-base font-normal (Form labels, table content)
- Small: text-sm (Helper text, timestamps, scan counts)
- Tiny: text-xs (Status badges, metadata)

## Layout System

**Spacing Primitives**: Use Tailwind units of **2, 4, 6, 8, 12, 16** exclusively
- Component padding: p-6 or p-8
- Section spacing: mb-8, mt-12
- Form field gaps: gap-6
- Card spacing: space-y-4

**Container Strategy**:
- Public pages: max-w-2xl mx-auto (centered, focused)
- Admin dashboard: max-w-7xl mx-auto (wide, data-optimized)
- Forms: max-w-lg for single-column layouts

## Component Library

### Public Registration Form
**Layout**: Single-page centered form with trust indicators
- Hero section (h-auto, not forced viewport):
  - Large heading + subheading
  - Event details card (date, venue, capacity) - 2-column grid on desktop
  - Trust badge: "Secure registration â€¢ Instant confirmation"
- Form section:
  - Single column, generous spacing (gap-6)
  - Floating labels or top-aligned labels with text-sm
  - Input fields: h-12, rounded-lg, border-2
  - Group size selector: Radio buttons in horizontal layout (up to 4)
  - Submit button: w-full, h-12, rounded-lg, font-semibold
- Confirmation section (post-registration):
  - Success icon (checkmark circle, large)
  - Confirmation message
  - Details summary card
  - "QR code will be generated by admin" notice

### Admin Dashboard

**Navigation**: 
- Top bar with app name, admin indicator, logout button
- Horizontal tabs: "Registrations" | "Generate QR" | "Scan Entry" | "Export"

**Registrations Table**:
- Compact data table with sticky header
- Columns: ID (monospace) | Name | Email | Phone | Organization | Group Size | Scans | Status | Actions
- Row height: h-16 for readability
- Alternating row treatment for scannability
- Status badges: inline-flex items-center px-3 py-1 rounded-full text-xs
  - "Pending QR" | "Active" | "Exhausted (4/4 scans)" | "Invalid"

**QR Generation View**:
- List/card hybrid showing pending registrations
- Each card: Registration details on left, "Generate QR" button on right
- Button: px-6 py-3 rounded-lg font-medium
- Generated state: Shows QR code preview (200x200px), download button, copy verification URL

**QR Scanner Interface**:
- Full-width camera viewfinder (aspect-ratio-square max-w-2xl mx-auto)
- Camera feed with overlay guide (rounded corners, scanning zone indicator)
- Results area below scanner:
  - Success state: Large checkmark, attendee name, scans remaining (e.g., "2/4 entries used")
  - Error state: X icon, error message ("Invalid QR" or "Maximum entries reached")
  - Visual feedback: Results card with h-32, flex items, rounded-lg, p-6
- Scan history sidebar (on desktop): Recent scans list with timestamps

**Export Section**:
- Filter controls: Date range picker, status filter (2-column grid)
- Export format buttons: "CSV" | "JSON" | "PDF Report"
- Summary cards row (3-column grid):
  - Total registrations
  - QR codes generated
  - Total entries logged

### Shared Components

**Forms**:
- Input fields: Consistent h-12, px-4, rounded-lg, border-2
- Focus states: border width increases, subtle shadow
- Error states: border treatment change, text-sm error message below with text-red-600
- Required indicators: Small asterisk, not prominent

**Buttons**:
- Primary: h-12, px-8, rounded-lg, font-semibold
- Secondary: h-10, px-6, rounded-lg, font-medium  
- Icon buttons (scanner controls): h-10 w-10, rounded-lg
- When on images: backdrop-blur-md treatment

**Cards**:
- Elevated cards: rounded-xl, shadow-md, p-6
- Flat cards (dashboard): rounded-lg, border-2, p-4
- Interactive cards: add hover shadow transition

**Badges/Pills**:
- Status badges: inline-flex items-center px-3 py-1 rounded-full text-xs font-medium
- Count badges: h-6 min-w-6 rounded-full flex items-center justify-center text-xs

## Responsive Breakpoints

**Mobile (base)**:
- Single column layouts
- Full-width buttons
- Stacked form fields
- Hide secondary dashboard columns
- Simplified scanner interface

**Tablet (md:)**:
- 2-column form sections where logical
- Show more table columns
- Side-by-side scanner + results

**Desktop (lg:)**:
- Full table display
- Multi-column dashboard layout
- Scan history sidebar visible

## Accessibility

- All form inputs have associated labels (visible or aria-label)
- Focus indicators on all interactive elements (2px offset ring)
- Sufficient contrast for all text (WCAG AA minimum)
- Keyboard navigation support for scanner (spacebar to scan)
- Screen reader announcements for scan results
- Error messages clearly associated with fields

## Images

**Public Registration Page**:
- Hero background image: Event venue or abstract geometric pattern, full-width but natural height (not forced 100vh), subtle overlay for text readability
- Trust/security icon: Small illustrative element near form submission

**Admin Dashboard**:
- No decorative images
- QR code previews: 200x200px, centered in cards
- Icon set: Heroicons for all UI icons (outline style for navigation, solid for status indicators)

## Animation Guidelines

**Minimal, purposeful only**:
- Scan result transitions: 300ms fade-in for success/error states
- Table row hover: Subtle lift (1px translate-y)
- Button interactions: Standard hover/active states (no custom animations)
- QR generation: Simple fade-in of QR code (400ms)
- **No scroll-triggered animations, no elaborate transitions**
